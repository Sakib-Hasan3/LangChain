## Deploy LangServe Runnable and Chain as API

**LangServe** হলো এমন একটি library যা **LangChain runnables এবং chains-কে REST API হিসেবে expose** করতে দেয়। এটা **FastAPI**-এর ওপর built, **Pydantic** দিয়ে validation handle করে, আর deployed runnable-এ remote client দিয়েও call করা যায়। ([PyPI][1])

### এটা কেন দরকার

তুমি notebook-এ chain run করলে সেটা শুধু local demo।
কিন্তু real app-এ তোমার দরকার হয়:

* frontend থেকে call করার মতো API
* reusable endpoint
* validation
* deployment-ready serving layer

সেখানেই LangServe useful। এটা basically তোমার **`Runnable` বা `Chain` → HTTP endpoint**-এ convert করে। ([PyPI][1])

---

## Core idea

ধরো তোমার কাছে একটা LCEL chain আছে:

```python
chain = prompt | llm | parser
```

এখন তুমি চাইছ এটা browser, frontend, বা অন্য backend থেকে hit করা যাক।

LangServe দিয়ে সেটা সাধারণত এই flow-এ হয়:

```text
Runnable / Chain
      ↓
FastAPI app
      ↓
LangServe route
      ↓
REST API endpoint
```

এই setup LangServe-এর documented purpose-এর সাথেই মিলে: LangChain runnables-কে REST API হিসেবে deploy করা। ([PyPI][1])

## সাধারণ setup flow

### 1. Packages install

তোমার সাধারণত লাগবে:

```bash
pip install langserve fastapi uvicorn
```

LangServe নিজেই FastAPI integration-এর জন্য designed। ([PyPI][1])

---

### 2. একটি runnable বা chain বানাও

উদাহরণ:

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template("Explain {topic} simply.")
llm = ChatOpenAI(model="gpt-4.1-mini", temperature=0)
parser = StrOutputParser()

chain = prompt | llm | parser
```

এখানে `chain`-টাই পরে API হবে।

---

### 3. FastAPI app তৈরি করো

```python
from fastapi import FastAPI
from langserve import add_routes

app = FastAPI(
    title="My LangChain API",
    version="1.0",
    description="Simple LangServe API"
)

add_routes(app, chain, path="/explain")
```

### এটা কী করল

`add_routes(...)` দিয়ে chain-টাকে app-এর মধ্যে route হিসেবে attach করা হলো।
LangServe-এর মূল কাজই এই runnable serving layer দেওয়া। ([PyPI][1])

---

### 4. Server চালাও

```bash
uvicorn app:app --reload
```

এটা FastAPI server চালাবে।

তারপর সাধারণত local server থেকে endpoint serve হবে, আর তুমি সেটা frontend বা API client থেকে hit করতে পারবে। FastAPI + LangServe deployment pattern এইভাবেই work করে। ([PyPI][1])

---

## Minimal full example

```python
from fastapi import FastAPI
from langserve import add_routes

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template("Explain {topic} in simple words.")
llm = ChatOpenAI(model="gpt-4.1-mini", temperature=0)
parser = StrOutputParser()

chain = prompt | llm | parser

app = FastAPI(
    title="LangServe Demo",
    version="1.0",
    description="Deploying a runnable as API"
)

add_routes(app, chain, path="/chain")
```

Run:

```bash
uvicorn app:app --reload
```

## কী expose হয়

LangServe deployed runnable-কে API হিসেবে call করার জন্য serve করে, আর এর সাথে client-side integration-ও available থাকে। PyPI description অনুযায়ী JavaScript client-ও আছে LangChain.js ecosystem-এ। ([PyPI][2])

এর মানে:

* Python backend থেকে call করা যায়
* JS frontend থেকেও call করা যায়
* same chain reusable service হয়ে যায়

## কোথায় deploy করবে

Local testing-এর জন্য `uvicorn` যথেষ্ট।
Production-এর জন্য তুমি যেকোনো standard FastAPI-compatible environment-এ deploy করতে পারো, যেমন:

* VM
* Docker container
* cloud app platform

এখানে brutal truth হলো: **LangServe deployment মানে শুধু code expose করা, scale/security/monitoring magically solve হয় না.**
