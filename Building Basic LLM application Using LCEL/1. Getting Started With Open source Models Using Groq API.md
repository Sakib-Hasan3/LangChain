## Getting Started With Open-Source Models Using Groq API

**Groq API** দিয়ে তুমি hosted **open-source models** খুব কম latency-তে ব্যবহার করতে পারো। শুরু করার minimum path হলো: API key তৈরি করা, `GROQ_API_KEY` environment variable set করা, Groq SDK বা OpenAI-compatible client ব্যবহার করা, তারপর একটি supported model দিয়ে প্রথম chat completion চালানো। Groq-এর official quickstart এই flow-টাই দেখায়। ([GroqCloud][1])

### Groq API আসলে কী

Groq API inference serve করে, আর official docs অনুযায়ী এটি **OpenAI-compatible endpoint**-ও দেয় at `https://api.groq.com/openai/v1`। তাই তুমি চাইলে Groq-এর own SDK ব্যবহার করতে পারো, অথবা existing OpenAI client config change করেও migrate করতে পারো। ([GroqCloud][2])

### কেন এটা useful

Open-source model local machine-এ চালানো যায়, কিন্তু সবসময় practical না। Groq hosted inference দেয়, তাই:

* heavy hardware লাগে না
* OpenAI-style API pattern পাওয়া যায়
* supported open models দিয়ে দ্রুত prototype বানানো যায় ([GroqCloud][1])

## Step 1: API key তৈরি করো

Groq quickstart অনুযায়ী আগে console থেকে API key create করতে হবে, তারপর shell-এ এটা environment variable হিসেবে set করা best practice: ([GroqCloud][1])

```bash
export GROQ_API_KEY=your_api_key_here
```

Windows PowerShell হলে সাধারণত এভাবে:

```powershell
$env:GROQ_API_KEY="your_api_key_here"
```

---

## Step 2: Python SDK install করো

Groq quickstart Python-এর জন্য `groq` package install করতে বলে। ([GroqCloud][1])

```bash
pip install groq
```

---

## Step 3: প্রথম model call চালাও

Groq-এর official example অনুযায়ী basic chat completion এভাবে করা যায়: ([GroqCloud][1])

```python
import os
from groq import Groq

client = Groq(api_key=os.environ.get("GROQ_API_KEY"))

chat_completion = client.chat.completions.create(
    messages=[
        {
            "role": "user",
            "content": "Explain open-source language models in simple terms."
        }
    ],
    model="llama-3.3-70b-versatile",
)

print(chat_completion.choices[0].message.content)
```

---

## Step 4: কোন model use করবে

Groq official docs-এ **Supported Models** page আছে, যেখানে available model IDs, context window, limits, pricing, আর preview status দেখানো হয়। গুরুত্বপূর্ণ ব্যাপার: **preview models production-এর জন্য safe assumption না**, কারণ docs explicitly বলে এগুলো short notice-এ discontinued হতে পারে। তাই শুরুতে stable, documented model ID নাও; code-এ random blog-post model নাম বসিয়ে error খাওয়ার দরকার নেই। ([GroqCloud][3])

---

## Step 5: চাইলে OpenAI client দিয়েও Groq ব্যবহার করো

Groq docs অনুযায়ী OpenAI compatibility আছে। শুধু `base_url` বদলাও, আর Groq API key দাও। ([GroqCloud][4])

```python
import os
import openai

client = openai.OpenAI(
    base_url="https://api.groq.com/openai/v1",
    api_key=os.environ.get("GROQ_API_KEY"),
)

response = client.chat.completions.create(
    model="llama-3.3-70b-versatile",
    messages=[
        {"role": "user", "content": "What is Groq API?"}
    ],
)

print(response.choices[0].message.content)
```

এটা useful যদি তুমি আগে থেকেই OpenAI-style codebase ব্যবহার করো।

---

## কিছু limitations মাথায় রাখো

Groq-এর OpenAI compatibility page অনুযায়ী সব OpenAI features 1:1 supported না। উদাহরণ হিসেবে কিছু unsupported fields আছে, যেমন `logprobs`, `logit_bias`, `top_logprobs`, আর `messages[].name`—এগুলো পাঠালে error আসতে পারে। তাই “OpenAI-compatible” মানে “exactly identical” ধরে নেওয়া বোকামি। ([GroqCloud][4])

## Minimal mental model

তোমার flow আসলে এটা:

```text
Prompt → Groq API → Open-source model → Response
```

আর production mindset হলে:

```text
Prompt → Validation → Model selection → API call → Response handling → Logging
```

---

## Brutal truth

শুধু Groq API call করতে পারা কোনো skill না। এটা trivial. আসল skill হলো:

* সঠিক model বেছে নেওয়া
* prompt ঠিক করা
* latency বনাম quality tradeoff বোঝা
* unsupported features এড়িয়ে চলা
* evaluation করা

যদি তুমি শুধু “API call কাজ করছে” stage-এ থাকো, তুমি এখনও builder না — তুমি setup phase-এ আছ।
