
## 4. Creating Streamlit Web App and Integrating Response With OpenAI API — বাংলা ব্যাখ্যা

**Streamlit**-এ chat app বানানোর জন্য `st.chat_input` আর `st.chat_message` built-in chat components আছে। OpenAI-এর current Python SDK-এ text generation-এর primary interface হলো **Responses API**। আর LangChain side-এ OpenAI chat models use করতে `langchain-openai` package লাগে। ([Streamlit Docs][1])

---

## 1. Streamlit web app কী?

Streamlit হলো Python দিয়ে দ্রুত web app বানানোর framework। Latest docs অনুযায়ী install command `pip install streamlit`, আর app run করতে `streamlit run your_file.py` use করা হয়। ([Streamlit Docs][2])

### কেন এটা beginner-friendly?

কারণ:

* HTML/CSS/JS লাগে না basic level-এ
* pure Python দিয়েই UI বানানো যায়
* chat interface ready-made components দিয়ে করা যায়

---

## 2. OpenAI API integration মানে কী?

এখানে “integration” মানে হলো:

* user Streamlit-এ input দেবে
* app সেটা OpenAI model-এ পাঠাবে
* response নিয়ে UI-তে দেখাবে

OpenAI official quickstart অনুযায়ী Python-এ `openai` SDK install করে client বানানো হয়, এবং Responses API দিয়ে request পাঠানো হয়। Model choose করার সময় official models page বলছে complex reasoning/coding-এর জন্য `gpt-5.4` start point হিসেবে ধরা যায়, আর lower latency/cost-এর জন্য smaller variants আছে। ([OpenAI Developers][3])

---

## 3. এই topic-এর সবচেয়ে important mental model

এটা মাথায় বসাও:

> **Streamlit = app-এর মুখ**
> **OpenAI API = brain-এর access**
> **তোমার code = দুইটার মধ্যে bridge**

তুমি যদি ভাবো শুধু `st.chat_input()` দিলেই AI app হয়ে গেল, তাহলে তুমি UI দেখছ, system না।


## 6. সবচেয়ে simple Streamlit + OpenAI app

নিচের app-টা beginner-friendly, এবং direct OpenAI SDK use করে।

### `app.py`

```python
import os
from openai import OpenAI
import streamlit as st
from dotenv import load_dotenv

load_dotenv()

st.set_page_config(page_title="OpenAI Chat App", page_icon="💬")
st.title("OpenAI Chat App")

api_key = os.getenv("OPENAI_API_KEY")
if not api_key:
    st.error("OPENAI_API_KEY পাওয়া যায়নি। .env file বা environment variable সেট করো।")
    st.stop()

client = OpenAI(api_key=api_key)

# Chat history
if "messages" not in st.session_state:
    st.session_state.messages = []

# পুরনো messages show করো
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# নতুন input নাও
user_prompt = st.chat_input("তোমার প্রশ্ন লিখো...")

if user_prompt:
    # user message save
    st.session_state.messages.append({"role": "user", "content": user_prompt})

    with st.chat_message("user"):
        st.markdown(user_prompt)

    try:
        with st.chat_message("assistant"):
            with st.spinner("উত্তর তৈরি হচ্ছে..."):
                response = client.responses.create(
                    model="gpt-5.4-mini",
                    input=user_prompt
                )

                answer = response.output_text
                st.markdown(answer)

        # assistant message save
        st.session_state.messages.append({"role": "assistant", "content": answer})

    except Exception as e:
        st.error(f"Error: {e}")
```

### Run command

```bash
streamlit run app.py
```

OpenAI official docs অনুযায়ী `client.responses.create(...)` হচ্ছে current standard Python pattern, আর Streamlit docs অনুযায়ী `st.chat_input` ও `st.chat_message` chat UI building-এর জন্য intended components। ([OpenAI Developers][3])

## 7. এই code-এ কী হচ্ছে?

### `st.chat_input()`

এটা user-এর কাছ থেকে chat-style input নিচ্ছে। Streamlit docs-এ এটা conversational input widget হিসেবে documented। ([Streamlit Docs][5])

### `st.chat_message()`

এটা user/assistant message UI-তে render করছে। Streamlit docs অনুযায়ী chat containers-এর ভিতরে markdown, tables, charts—সবই রাখা যায়। ([Streamlit Docs][1])

### `st.session_state`

এটা chat history store করছে, যাতে rerun হলেও conversation হারিয়ে না যায়। Streamlit tutorials-এ chat app state management-এর জন্য session state use করা হয়। ([Streamlit Docs][6])

### `client.responses.create(...)`

এখানে OpenAI model-এ prompt পাঠানো হচ্ছে। OpenAI docs অনুযায়ী Responses API হলো unified interface for text generation and agent-like flows। ([OpenAI Developers][7])

## 9. Better version: conversation history সহ

যদি তুমি previous messages model-এ পাঠাতে চাও, direct SDK দিয়ে সেটা structure করে পাঠাতে পারো। Responses API multi-turn interactions support করে। ([OpenAI Developers][7])

```python
import os
from openai import OpenAI
import streamlit as st
from dotenv import load_dotenv

load_dotenv()

st.title("Conversation App")

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

if "messages" not in st.session_state:
    st.session_state.messages = [
        {"role": "system", "content": "You are a helpful AI assistant. Reply in Bangla."}
    ]

for msg in st.session_state.messages:
    if msg["role"] != "system":
        with st.chat_message(msg["role"]):
            st.markdown(msg["content"])

prompt = st.chat_input("প্রশ্ন করো...")

if prompt:
    st.session_state.messages.append({"role": "user", "content": prompt})

    with st.chat_message("user"):
        st.markdown(prompt)

    with st.chat_message("assistant"):
        with st.spinner("ভাবছি..."):
            response = client.responses.create(
                model="gpt-5.4-mini",
                input=st.session_state.messages
            )
            answer = response.output_text
            st.markdown(answer)

    st.session_state.messages.append({"role": "assistant", "content": answer})
```

## 10. LangChain দিয়ে একই idea

তুমি যদি Streamlit + LangChain + OpenAI combine করতে চাও, pattern হবে এভাবে:

```python
import os
import streamlit as st
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage

load_dotenv()

st.title("LangChain + OpenAI Chatbot")

if "messages" not in st.session_state:
    st.session_state.messages = [
        SystemMessage(content="You are a helpful assistant. Reply in Bangla.")
    ]

llm = ChatOpenAI(
    model="gpt-5.4-mini",
    temperature=0.3,
    api_key=os.getenv("OPENAI_API_KEY")
)

for msg in st.session_state.messages:
    if isinstance(msg, HumanMessage):
        with st.chat_message("user"):
            st.markdown(msg.content)
    elif isinstance(msg, AIMessage):
        with st.chat_message("assistant"):
            st.markdown(msg.content)

prompt = st.chat_input("তোমার প্রশ্ন লিখো...")

if prompt:
    human_msg = HumanMessage(content=prompt)
    st.session_state.messages.append(human_msg)

    with st.chat_message("user"):
        st.markdown(prompt)

    with st.chat_message("assistant"):
        with st.spinner("উত্তর তৈরি হচ্ছে..."):
            response = llm.invoke(st.session_state.messages)
            st.markdown(response.content)

    st.session_state.messages.append(AIMessage(content=response.content))
```

LangChain docs অনুযায়ী OpenAI chat models-এর জন্য `ChatOpenAI` হলো standard wrapper, আর LangChain chat models message-based invocation support করে। ([LangChain Docs][4])

## 11. Direct OpenAI বনাম LangChain

| Approach           | কবে use করবে               | সুবিধা                | downside         |
| ------------------ | -------------------------- | --------------------- | ---------------- |
| Direct OpenAI SDK  | simple app                 | কম complexity         | orchestration কম |
| LangChain + OpenAI | chain, memory, tools লাগবে | scalable architecture | বেশি abstraction |

OpenAI SDK direct use করলে control বেশি, layer কম। LangChain useful হয় যখন prompt templates, tools, memory, agents, structured chaining দরকার হয়। এটা LangChain overview এবং model integration docs-এর সাথে consistent। ([OpenAI Developers][3])

## 12. Streaming কেন important?

Streamlit docs chat apps-এর জন্য long-running process status দেখাতে `st.status` বা progressive UI patterns suggest করে, আর OpenAI/ LangChain docs streaming support-এর কথা বলে। Streaming দিলে user full answer-এর জন্য dead wait করে না। ([Streamlit Docs][8])

Brutal truth:
Streaming ছাড়া তোমার app technically কাজ করলেও UX weak লাগবে।

## 13. Common mistakes

### 1. API key code-এ hardcode করা

খারাপ habit। Environment variable use করো। ([OpenAI Developers][3])

### 2. Streamlit rerun model না বোঝা

প্রতি interaction-এ script rerun হয়। তাই `st.session_state` critical। Streamlit tutorials state-driven flow ব্যবহার করে। ([Streamlit Docs][6])

### 3. Chat history না রাখা

তাহলে app every turn-এ memoryless হয়ে যায়।

### 4. Wrong model blindly use করা

Official models page অনুযায়ী model choice cost/latency/capability tradeoff-এর উপর নির্ভর করে। ([OpenAI Developers][9])

### 5. Deprecated patterns copy করা

OpenAI official direction এখন Responses API-এর দিকে; Assistants API deprecated এবং August 26, 2026-এ shutdown scheduled। ([OpenAI Developers][7])
