## 6. **Implementing Tools in LangChain** — বাংলা ব্যাখ্যা

এই topic-এ অনেকে সবচেয়ে বেশি ভুল করে।
কারণ তারা ভাবে **tool মানে শুধু function**।
না। LangChain-এ tool হলো **LLM-এর জন্য controllable capability** — মানে model প্রয়োজন হলে কোনো function call করতে পারবে, input দেবে, result নেবে, তারপর সেটা ব্যবহার করে উত্তর বানাবে। LangChain-এর অফিসিয়াল docs অনুযায়ী tools agent-কে real-time data fetch করা, code execute করা, database query করা, বা external action নেওয়ার ক্ষমতা দেয়। ([LangChain Docs][1])

---

## 1. Tool কী?

সহজ ভাষায়:

> **Tool = এমন function যেটা model প্রয়োজন হলে call করতে পারে**

উদাহরণ:

* calculator
* weather checker
* web search
* database lookup
* custom business logic

LLM নিজে real-world action নিতে পারে না। Tool সেই gap fill করে। অফিসিয়াল docs-এও বলা আছে tools হচ্ছে callable functions with well-defined inputs and outputs, যেগুলো chat model-এর কাছে pass করা হয়, আর model conversation context দেখে decide করে কখন tool invoke করবে এবং কী arguments দেবে। ([LangChain Docs][1])

---

## 2. কেন tools দরকার?

কারণ pure LLM:

* current weather জানে না
* live database read করতে পারে না
* secure internal API call করতে পারে না
* deterministic calculation সবসময় reliable ভাবে করে না

তাই tools implement করতে হয়, যাতে model:

1. বুঝতে পারে কোন কাজের জন্য কোন tool লাগবে
2. structured input পাঠাতে পারে
3. result নিয়ে final answer দিতে পারে

এটাই agent-এর core power। Agents docs-এ LangChain স্পষ্টভাবে tools-কে agent execution-এর কেন্দ্রে রেখেছে। ([LangChain Docs][2])

---

## 3. Tool implement করার সবচেয়ে common উপায়

সবচেয়ে standard উপায় হলো `@tool` decorator ব্যবহার করা।

### Basic example

```python
from langchain.tools import tool

@tool
def multiply(a: int, b: int) -> int:
    """Multiply two numbers."""
    return a * b
```

এই style LangChain docs-এ directly দেখানো হয়েছে। সেখানে arithmetic tools যেমন `add`, `multiply`, `divide` `@tool` decorator দিয়ে define করা হয়েছে। ([LangChain Docs][3])

---

## 4. এই code-এ আসলে কী হলো?

উপরের function normal Python function না আর। এটা এখন LangChain-readable tool।

### LangChain কী infer করে?

* tool-এর **name**
* tool-এর **description**
* tool-এর **arguments schema**
* input type
* output handling

Agents docs অনুযায়ী `tool` decorator দিয়ে tool name, description, argument schema ইত্যাদি customize করা যায়। ([LangChain Docs][2])

---

## 5. Docstring কেন critical?

এই জায়গাতেই beginnerরা গণ্ডগোল করে।

```python
@tool
def multiply(a: int, b: int) -> int:
    """Multiply two numbers."""
    return a * b
```

এখানে `"Multiply two numbers."` শুধু decoration না।
এই description model-কে বোঝায় toolটা **কখন ব্যবহার করতে হবে**।

যদি docstring vague হয়, model wrong tool pick করতে পারে।

### খারাপ example

```python
@tool
def process(x, y):
    """Does something."""
    return x * y
```

এটা garbage।
LLM বুঝবে না এটা calculator, formatter, না অন্য কিছু।

---

## 6. Tool implement করার mental model

প্রতিটা ভালো tool-এর ৪টা জিনিস পরিষ্কার হওয়া দরকার:

### A. **Name**

Tool কী কাজ করে

### B. **Description**

কখন use করতে হবে

### C. **Inputs**

কি arguments লাগবে

### D. **Output**

কি return করবে

এই structure না থাকলে tool calling unreliable হবে। LangChain docs-এর tool examples-এ typed arguments এবং explicit descriptions consistently ব্যবহার করা হয়েছে। ([LangChain Docs][1])

---

## 7. Tool agent-এর সাথে কীভাবে যুক্ত হয়?

একটা agent-এ tools list আকারে pass করা হয়।

### Example

```python
from langchain.tools import tool
from langchain.chat_models import init_chat_model
from langchain.agents import create_agent

@tool
def add(a: int, b: int) -> int:
    """Add two integers."""
    return a + b

model = init_chat_model("claude-sonnet-4-6", temperature=0)

agent = create_agent(
    model=model,
    tools=[add]
)
```

LangChain agents docs-এ static tools list আকারে agent-এ pass করার pattern দেখানো হয়েছে, এবং plain Python function বা coroutine-ও tool হিসেবে ব্যবহার করা যায় বলা হয়েছে। ([LangChain Docs][2])

---

## 8. Modern tool binding pattern

আরও modern pattern হলো model-এর সাথে tools bind করা।

### Example idea

```python
model_with_tools = model.bind_tools(tools)
```

LangGraph quickstart-এ official example-এ `model.bind_tools(tools)` ব্যবহার করে model-কে tools-aware বানানো হয়েছে, তারপর model response-এর `tool_calls` inspect করে tool node execute করা হয়েছে। ([LangChain Docs][3])

---

## 9. Tool call execution flow

একটা tool-enabled system মোটামুটি এভাবে কাজ করে:

```text
User input
   ↓
LLM দেখে tool দরকার কিনা
   ↓
Tool call generate করে
   ↓
Actual tool function run হয়
   ↓
Result model-এর কাছে ফেরত যায়
   ↓
Model final answer দেয়
```

LangGraph quickstart-এ এই exact loop দেখানো হয়েছে: model call → check `tool_calls` → tool invoke → `ToolMessage` ফেরত → model আবার call। ([LangChain Docs][3])

---

## 10. Tool node কী?

যদি তুমি agent loop manually control করো, তখন একটা **tool node** বা tool execution block দরকার হয়।

Docs-এর example-এ:

* model শেষ message-এ `tool_calls` দেয়
* code সেই tool name lookup করে
* `tool.invoke(...)` চালায়
* observation `ToolMessage` হিসেবে ফেরত পাঠায়

এটা দেখায় যে tools শুধু declare করলেই শেষ না — **execution path**-ও বুঝতে হবে। ([LangChain Docs][3])

## 11. Practical custom tool example

ধরো তুমি user-এর name length count করার tool বানাতে চাও:

```python
from langchain.tools import tool

@tool
def get_name_length(name: str) -> int:
    """Return the number of characters in a person's name."""
    return len(name)
```

এখন model যদি জিজ্ঞেস পায়:

> “Rahim নামের length কত?”

তাহলে theoretically এই tool use করতে পারবে।

## 14. Good tool design rules

### 1. এক tool = এক clear responsibility

একটা tool-এ ১০টা কাজ গুঁজে দিও না।

### 2. Typed arguments দাও

`a: int, b: int` style রাখো।

### 3. ভালো docstring লিখো

LLM-এর decision quality এটার উপর নির্ভর করে।

### 4. Deterministic output রাখো

Tool unpredictable হলে model confused হবে।

### 5. Short, clean return value দাও

Messy HTML dump বা giant blob দিলে reasoning degrade করতে পারে।

## 15. Beginners কোথায় fail করে?

### ❌ Tool description useless

“Does task”, “Helps user”, “Processes data” — এগুলো worthless।

### ❌ Too much logic in one tool

একটা tool data fetch + summarize + format + classify সব একসাথে করছে।
এতে tool selection harder হয়।

### ❌ Untyped inputs

LLM structured args দিতে গেলে schema clear দরকার।

### ❌ No error handling

API fail করলে কী হবে? empty result এলে কী হবে?

### ❌ সবকিছু tool বানানো

যা normal chain-এ solve হয়, তা tool বানিয়ে complexity বাড়িও না।

---

## 16. Implementing tools ≠ importing tools

এখানে একটা mental trap আছে।

অনেকে ভাবে:

* library থেকে tool import করলাম
* list-এ দিলাম
* কাজ শেষ

না।

**Implementing tools** মানে:

* tool design করা
* description ঠিক করা
* input/output shape ঠিক করা
* agent-এর সাথে bind করা
* execution flow বুঝা
* failure mode handle করা

## 17. Minimal end-to-end example

```python
from langchain.tools import tool
from langchain.chat_models import init_chat_model

@tool
def multiply(a: int, b: int) -> int:
    """Multiply two integers."""
    return a * b

tools = [multiply]

model = init_chat_model("claude-sonnet-4-6", temperature=0)
model_with_tools = model.bind_tools(tools)

response = model_with_tools.invoke("What is 6 multiplied by 7?")
print(response)
```

এই pattern official docs-এর bind-tools direction-এর সাথে aligned। তবে practical agent loop-এ model যদি tool call দেয়, সেই tool call execute করে result ফেরত দেওয়ার logicও লাগবে। ([LangChain Docs][3])

## 18. Agent loop ছাড়া tool calling incomplete

এটাই hidden truth।

তুমি যদি শুধু `bind_tools()` করো, তারপর ভাবো সব শেষ — ভুল।
Model tool call suggest করতে পারে, কিন্তু tool চালিয়ে result ফেরত দেওয়ার loop না থাকলে complete agent behavior আসবে না। অফিসিয়াল LangGraph quickstart এই loop explicitly দেখায়। ([LangChain Docs][3])

## 20. Exam-style short note

**Implementing Tools in LangChain** মানে:

* Python function-কে tool আকারে define করা
* proper name, description, inputs দেওয়া
* tools-কে model বা agent-এর সাথে integrate করা

## 21. Brutal truth

তুমি যদি tools-কে শুধু “extra functions” ভাবো, তাহলে LangChain-এর অর্ধেকই বুঝনি।

Tools আসলে:

* capability boundary
* control interface
* reasoning extension layer

Agent-এর intelligence অনেক সময় model-এর চেয়ে বেশি নির্ভর করে **tool design quality**-এর উপর।

খারাপ tool design = খারাপ agent
এই সত্যটা beginners accept করতে চায় না।
