## 12. **Human-in-the-Loop (HITL) Middleware Using LangChain** — বাংলা ব্যাখ্যা

এই topicটা শুধু feature না — এটা **AI safety + control layer**।
কারণ agent যদি tool call করতে পারে, কিন্তু oversight না থাকে, তাহলে:

* ভুল email send করতে পারে
* ভুল SQL query চালাতে পারে
* file delete করতে পারে
* sensitive action নিতে পারে

**LangChain**-এর **Human-in-the-Loop middleware** agent-এর tool calls pause করে human approval, edit, বা reject করার সুযোগ দেয়। এটা tool calls-কে policy অনুযায়ী check করে, interrupt trigger করে, state save করে, তারপর human decision-এর পর resume করে। ([LangChain Docs][1])

---

# 🔷 1. HITL Middleware কী?

সহজভাবে:

> **Agent কোনো sensitive action নেওয়ার আগে মানুষের approval নেওয়ার system**

মানে:

* agent tool call suggest করলো
* middleware check করলো
* risky হলে pause
* human approve/edit/reject করলো
* তারপর agent continue করলো

---

# 🔷 2. কেন দরকার?

LLM agent smart হলেও:

* hallucinate করতে পারে
* wrong action নিতে পারে
* context misread করতে পারে

> “পুরনো temp.txt file delete করো”

Agent:

👉 middleware না থাকলে:

* file instantly delete

👉 middleware থাকলে:

* human confirmation চাইবে

# 🔷 4. LangChain-এ HITL Middleware কীভাবে কাজ করে?

Flow:

```text
User input
   ↓
LLM tool call propose করে
   ↓
Middleware policy check
   ↓
Sensitive?
   ↓
Yes → Interrupt / Pause
   ↓
Human decision
   ↓
Resume execution
   ↓
Final answer
```

LangChain docs অনুযায়ী HITL middleware tool call execution-এর আগে interrupt issue করে, আর graph state persistence layer-এ save হয়, যাতে safely later resume করা যায়। ([LangChain Docs][1])

# 🔷 5. Basic Code Example

```python
from langchain.agents import create_agent
from langchain.agents.middleware import HumanInTheLoopMiddleware
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="openai:gpt-4o",
    tools=[send_email_tool, delete_file_tool],
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "send_email_tool": True,
                "delete_file_tool": True
            }
        )
    ],
    checkpointer=InMemorySaver()
)
```

এই built-in middleware specific tool calls interrupt করতে পারে। `True` দিলে approve / edit / reject decisions allow হয়; `False` হলে tool auto-approved। ([LangChain Reference][2])

# 🔷 6. Code Breakdown

---

## ✅ `interrupt_on`

এটা policy define করে:

```python
interrupt_on={
    "send_email_tool": True
}
```

মানে:

* send_email_tool → human approval লাগবে

---

## ✅ `checkpointer`

Critical।

কারণ:

* pause হলে state save করতে হবে
* later resume করতে হবে

Without persistence:

* agent context হারাবে

LangChain v1-এর durable execution LangGraph-এর persistence-এর উপর built। ([LangChain Docs][3])

# 🔷 7. Human Decisions কী কী?

HITL middleware support করে:

### 1. Approve

Tool execute হবে

### 2. Reject

Tool skip / cancel

### 3. Edit

Tool arguments change করে approve

LangChain API reference-এ edit / approve / reject flows supported বলে উল্লেখ আছে। ([LangChain Reference][2])

# 🔷 11. Common Mistakes

---

## ❌ সব tool block করা

এতে agent useless হয়ে যায়

### Better:

* only risky tools gate করো

---

## ❌ no persistence

pause → state lost

---

## ❌ bad approval UX

human clear info না পেলে decision bad হবে

### show:

* tool name
* arguments
* risk

---

## ❌ too much trust in model

LLM “probably right” মানে safe না
