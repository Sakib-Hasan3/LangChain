## 11. **Summarization Middleware Using LangChain** — বাংলা ব্যাখ্যা

এই topicটা আসলে production-level AI system-এর backbone।
কারণ long conversation handle করতে না পারলে:

* token limit hit করবে
* cost বাড়বে
* latency বাড়বে
* model distracted হবে

LangChain-এর built-in **SummarizationMiddleware** conversation history-এর token count monitor করে, limit-এর কাছে গেলে পুরনো messages summarize করে, recent context intact রাখে। ([LangChain Reference][1])

---

# 🔷 1. Summarization Middleware কী?

সহজভাবে:

> **Long chat history ছোট করে compress করার automated layer**

মানে:

* পুরনো messages → summary
* নতুন recent messages → full form

এতে agent context হারায় না, আবার token overload-ও হয় না।

---

# 🔷 2. Middleware কী?

Middleware হলো LangChain agent loop-এর control layer।

এটা:

* model call-এর আগে কিছু করতে পারে
* model call-এর পরে কিছু করতে পারে

SummarizationMiddleware মূলত **before_model hook** use করে।
মানে model run হওয়ার আগেই history optimize করে। ([LangChain Docs][2])

---

# 🔷 3. কেন Summarization Middleware দরকার?

## Problem: Long conversation = bad performance

ধরো chatbot-এ ২০০ message হয়েছে।

যদি সব message model-এ পাঠাও:

* cost বাড়বে
* speed কমবে
* stale context model-কে confuse করবে

LangChain docs short-term memory section-এও বলেছে, long context model-কে distract করে এবং performance degrade করে। ([LangChain Docs][3])

---

## Solution:

Middleware:

* old history summarize করবে
* important flow retain করবে
* recent messages preserve করবে

---

# 🔷 4. কীভাবে কাজ করে?

Flow:

```text
User input
   ↓
Check token count
   ↓
Threshold exceed?
   ↓
Yes → summarize old messages
   ↓
Keep recent messages
   ↓
Send optimized context to LLM
```

LangChain reference অনুযায়ী, middleware AI/Tool message pairs intact রাখে যাতে execution trace না ভাঙে। ([LangChain Reference][1])

---

# 🔷 5. Basic Code Example

```python
from langchain.agents import create_agent
from langchain.agents.middleware import SummarizationMiddleware
from langgraph.checkpoint.memory import InMemorySaver

checkpointer = InMemorySaver()

agent = create_agent(
    model="openai:gpt-4o",
    tools=[],
    middleware=[
        SummarizationMiddleware(
            model="openai:gpt-4o-mini",
            max_tokens_before_summary=4000,
            messages_to_keep=20
        )
    ],
    checkpointer=checkpointer
)
```

এই pattern LangChain v1 docs-এর built-in middleware examples-এর সাথে aligned। ([LangChain Docs][4])

# 🔷 6. Code Breakdown

---

## ✅ `model="openai:gpt-4o-mini"`

এটা summary তৈরির model।

### Important:

summary model main model থেকে আলাদা হতে পারে।

### Smart practice:

* main task → expensive model
* summary → cheap model

👉 cost save হবে

---

## ✅ `max_tokens_before_summary`

কখন summarize করবে।

Example:

* 4000 token exceed → trigger

---

## ✅ `messages_to_keep`

শেষ কয়টা message full রাখবে।

Example:

* last 20 message untouched

---

## ✅ `checkpointer`

Conversation state store করতে লাগে।

Without checkpointer:

* history থাকবে না
* middleware context হারাবে

# 🔷 9. Common Mistakes

---

## ❌ সব message রেখে দেওয়া

Beginner trap:

> “সব context important”

ভুল।

সব context useful না।

---

## ❌ খুব aggressive summary

যদি খুব বেশি compress করো:

* nuance হারাবে
* tool trace হারাবে

---

## ❌ system prompt হারানো

এটা critical।

LangChain community-তে issue report হয়েছে যে summarization যদি system message remove করে, agent instructions হারাতে পারে। ([GitHub][5])

### Rule:

system prompt preserve করো

---

## ❌ wrong threshold

Too low:

* unnecessary summaries

Too high:

* late trigger

---

# 🔷 10. Summarization vs Memory

| Feature | Summarization         | Long-term Memory |
| ------- | --------------------- | ---------------- |
| Goal    | compress current chat | persist info     |
| Scope   | session               | across sessions  |
| Cost    | lower                 | depends          |

👉 Summarization middleware = short-term memory optimization

# 🔷 13. Custom Summary Prompt

তুমি চাইলে summary prompt customize করতে পারো:

    model="openai:gpt-4o-mini",
