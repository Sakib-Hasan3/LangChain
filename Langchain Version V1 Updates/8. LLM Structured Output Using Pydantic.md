## ৮. LLM Structured Output — Pydantic দিয়ে

LLM থেকে plain text পাওয়া সহজ, কিন্তু বাস্তব অ্যাপ্লিকেশনে দরকার structured data — যেমন JSON object যার নির্দিষ্ট field আর type আছে। **Pydantic** দিয়ে এই কাজটা অনেক সহজ ও নির্ভরযোগ্য হয়।### কেন Pydantic দরকার?

LLM সাধারণত plain text দেয়। কিন্তু যদি তুমি চাও সে নির্দিষ্ট format-এ data দিক (যেমন একটা বইয়ের নাম, লেখক, প্রকাশ সাল — আলাদা field হিসেবে), তাহলে Pydantic দিয়ে একটা "ছাঁচ" বানিয়ে দিতে হয়। LLM সেই ছাঁচ মেনে JSON দেয়, আর Pydantic সেটা validate করে Python object বানায়।

---

### ধাপে ধাপে কোড

**ধাপ ১ — Pydantic দিয়ে Schema বানাও**

```python
from pydantic import BaseModel
from typing import Optional

class Book(BaseModel):
    title: str
    author: str
    year: int
    genre: Optional[str] = None
```

এখানে `Book` হলো আমাদের "ছাঁচ"। LLM কে বলব — এই format-এ উত্তর দাও।

---

**ধাপ ২ — OpenAI / Instructor দিয়ে LLM call করো**

সবচেয়ে সহজ উপায় হলো `instructor` লাইব্রেরি ব্যবহার করা:

```python
import instructor
from openai import OpenAI

client = instructor.from_openai(OpenAI())

book = client.chat.completions.create(
    model="gpt-4o-mini",
    response_model=Book,          # Pydantic model দিয়ে দাও
    messages=[
        {"role": "user", "content": "হুমায়ূন আহমেদের একটি বিখ্যাত বইয়ের তথ্য দাও"}
    ]
)

print(book.title)   # "মিসির আলি সমগ্র"
print(book.year)    # 1985  (integer, string নয়)
print(book.author)  # "হুমায়ূন আহমেদ"
```

`response_model=Book` দিলেই instructor বাকি কাজ করে — LLM কে সঠিক format-এ JSON দিতে বলে, তারপর সেটা `Book` object-এ convert করে।

**ধাপ ৩ — Nested ও Complex Structure**

```python
from pydantic import BaseModel, Field
from typing import List

class Author(BaseModel):
    name: str
    birth_year: int

class BookList(BaseModel):
    books: List[Book]
    total_count: int = Field(description="মোট বইয়ের সংখ্যা")

# LLM এখন একসাথে একাধিক বই দেবে, structured format-এ
result = client.chat.completions.create(
    model="gpt-4o-mini",
    response_model=BookList,
    messages=[{"role": "user", "content": "৩টি বাংলা উপন্যাসের তথ্য দাও"}]
)

for book in result.books:
    print(f"{book.title} — {book.year}")
```

**ধাপ ৪ — Validation দিয়ে Data Quality নিশ্চিত করো**

```python
from pydantic import BaseModel, field_validator

class Movie(BaseModel):
    title: str
    rating: float
    year: int

    @field_validator("rating")
    @classmethod
    def rating_must_be_valid(cls, v):
        if not (0 <= v <= 10):
            raise ValueError("রেটিং ০ থেকে ১০ এর মধ্যে হতে হবে")
        return v

    @field_validator("year")
    @classmethod
    def year_must_be_realistic(cls, v):
        if v < 1888 or v > 2030:
            raise ValueError("অবাস্তব বছর")
        return v
```

LLM যদি ভুল data দেয় (যেমন rating=15), Pydantic সাথে সাথে error দেবে।

### Instructor ছাড়া — OpenAI JSON Mode

Instructor না চাইলে OpenAI-এর built-in `response_format` ব্যবহার করা যায়:

```python
import json
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-mini",
    response_format={"type": "json_object"},
    messages=[
        {"role": "system", "content": "তুমি JSON format-এ উত্তর দেবে।"},
        {"role": "user", "content": "একটি বইয়ের তথ্য দাও: title, author, year"}
    ]
)

raw = json.loads(response.choices[0].message.content)
book = Book(**raw)   # Pydantic দিয়ে validate করো
```

### সারসংক্ষেপ

| পদ্ধতি | সুবিধা | অসুবিধা |
|---|---|---|
| `instructor` লাইব্রেরি | সবচেয়ে সহজ, automatic retry | extra dependency |
| OpenAI JSON mode | built-in, কোনো extra লাইব্রেরি নেই | শুধু JSON, Pydantic আলাদা |
| System prompt দিয়ে | সব LLM-এ কাজ করে | কম reliable |

সবচেয়ে ভালো অভ্যাস: `instructor` + `pydantic` একসাথে ব্যবহার করো। এতে type safety, auto-validation, আর clean Python object — তিনটাই পাবে।
