## LangChain দিয়ে Agent তৈরি করা — বাংলা ব্যাখ্যা

তুমি শুধু “agent” শব্দটা মুখস্থ করলে হবে না। আগে বুঝতে হবে **agent আসলে কী problem solve করে**। না হলে তুমি chain আর agent-এর পার্থক্যই গুলিয়ে ফেলবে।

---

## 1. Agent কী?

LangChain-এ **Agent** হলো এমন একটি system, যেটা:

* user-এর input বুঝে
* decide করে **কি করতে হবে**
* কোন **tool** ব্যবহার করতে হবে তা বেছে নেয়
* tool থেকে result নিয়ে
* final answer তৈরি করে

অর্থাৎ, এটা শুধু fixed flow না।
এটা **decision-making flow**।

### সহজভাবে:

একটা normal chain হলো:

> Input → Prompt → LLM → Output

আর Agent হলো:

> Input → LLM ভাবে → কোন tool লাগবে decide করে → tool run করে → result দেখে → আবার ভাবে → final answer দেয়

---

## 2. কেন Agent দরকার?

সব problem direct LLM দিয়ে solve হয় না।

ধরো user বলল:

* “আজকের weather বলো”
* “এই math solve করো”
* “database থেকে customer info আনো”
* “web search করে latest AI news বলো”

এখন pure LLM:

* weather জানে না
* real-time data জানে না
* calculator না
* database access নেই

তাই Agent দরকার, যেন সে:

* weather API use করে
* calculator tool চালায়
* database query করে
* search tool use করে

---

## 3. Agent আর Chain-এর পার্থক্য

| বিষয়             | Chain              | Agent                |
| ---------------- | ------------------ | -------------------- |
| Flow             | Fixed              | Dynamic              |
| Decision নেয়?    | না                 | হ্যাঁ                |
| Tool choose করে? | না                 | হ্যাঁ                |
| Use case         | simple prompt flow | complex task solving |

### Brutal truth:

অনেক beginner সবখানে agent ঢুকায়।
এটা ভুল।

যেখানে simple chain-এ কাজ হয়ে যায়, সেখানে agent use করা মানে:

* unnecessary complexity
* slower execution
* বেশি bug
* বেশি cost

**সব problem-এর solution agent না।**

## 5. Agent কীভাবে কাজ করে?

একটা common flow:

1. User question দিল
2. Agent দেখে এটা কি direct answerable?
3. যদি tool লাগে, tool select করে
4. Tool run হয়
5. Tool output agent-এর কাছে ফিরে আসে
6. Agent final answer তৈরি করে

### Example

User:

> “25 * 48 কত?”

Agent:

* বুঝল calculation দরকার
* calculator tool use করল
* result নিল
* user-কে দিল

আরেকটা:

> “Dhaka-এর current weather বলো”

Agent:

* weather tool use করল
* result fetch করল
* final answer দিল

## 7. Conceptual code structure

নিচে একটা basic pattern দেখো:

```python
from langchain_openai import ChatOpenAI
from langchain.agents import initialize_agent, AgentType
from langchain.tools import Tool

def calculator_tool(query):
    return str(eval(query))

tools = [
    Tool(
        name="Calculator",
        func=calculator_tool,
        description="Useful for solving math problems"
    )
]

llm = ChatOpenAI(model="gpt-4o-mini")

agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True
)

response = agent.run("What is 25 * 48?")
print(response)
```

---

## 8. এই code-এ কী হচ্ছে?

### `Tool`

এখানে একটা tool define করা হয়েছে:

* নাম: `Calculator`
* কাজ: math solve করা
* description: agent-কে বুঝতে help করে কখন এটা use করতে হবে

### `ChatOpenAI`

এটা model load করছে

### `initialize_agent(...)`

এটা agent setup করছে tools + LLM দিয়ে

### `AgentType.ZERO_SHOT_REACT_DESCRIPTION`

এটা পুরনো but famous pattern:

* LLM tool description পড়ে
* decide করে কোন tool use করবে

### `verbose=True`

এটা important, কারণ তুমি intermediate reasoning steps দেখতে পারবে

## 9. Modern LangChain style

LangChain evolve করছে। পুরনো `initialize_agent` approach অনেক জায়গায় legacy feel দেয়।
নতুন pattern-এ structured agent creation বেশি ব্যবহার হয়।

একটা newer pattern এর example:

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain_core.prompts import ChatPromptTemplate

llm = ChatOpenAI(model="gpt-4o-mini")

@tool
def multiply(a: int, b: int) -> int:
    """Multiply two numbers."""
    return a * b

tools = [multiply]

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

response = agent_executor.invoke({"input": "What is 12 times 7?"})
print(response)
```

## 11. `@tool` decorator কী?

এটা function-কে LangChain tool বানায়।

যেমন:

```python
@tool
def get_name_length(name: str) -> int:
    """Returns the length of a name."""
    return len(name)
```

এখন agent বুঝতে পারবে:

* tool-এর নাম কী
* input কী
* কাজ কী

### Critical point:

**Docstring খুব important**
কারণ agent tool-এর description থেকে decide করে tool use করবে কিনা।

যদি description weak হয়, agent wrong tool choose করতে পারে।

## 14. Agent বানাতে beginners কোথায় fail করে?

### 1. Tool description খারাপ লেখা

Agent tool বুঝতেই পারে না।

### 2. সবকিছু agent দিয়ে solve করতে চাওয়া

যেখানে simple function enough, সেখানে agent use করে mess বানায়।

### 3. No guardrails

Agent কে বেশি freedom দিলে nonsense action করতে পারে।

### 4. Current info expected without tools

LLM alone real-time data জানে না—এই basic truth ignore করে।

### 5. Debug না করা

`verbose=True` না দিলে agent কী করছে তা বোঝা যায় না।
