## ধাপ ২: ডেটা প্রিপ্রসেসিং

```
কাঁচা টেক্সট → পরিষ্কার করা → টোকেনাইজেশন → ট্রেইনিংয়ের জন্য প্রস্তুত
```

### এখানে কী হয়:
```
১. ডুপ্লিকেট ডেটা বাদ দেওয়া
২. নিম্নমানের কন্টেন্ট ফিল্টার করা
৩. ক্ষতিকর/বিষাক্ত কন্টেন্ট মুছে ফেলা
৪. টোকেনাইজেশন (টেক্সট → সংখ্যা)

উদাহরণ:
  "I love AI" → [40, 1567, 9552]
  "আমি AI ভালোবাসি" → [234, 1567, 8823]
```

### টোকেনাইজেশন পদ্ধতি:
```
📌 BPE (Byte Pair Encoding)      → GPT মডেলে ব্যবহৃত
📌 SentencePiece                  → LLaMA মডেলে ব্যবহৃত
📌 WordPiece                      → BERT মডেলে ব্যবহৃত
```

## ধাপ ৩: প্রি-ট্রেইনিং (সবচেয়ে ব্যয়বহুল ধাপ)

### 🎯 উদ্দেশ্য: পরবর্তী টোকেন অনুমান করা

```
ইনপুট:  "বিড়ালটি চেয়ারের উপর ___"
আউটপুট: "বসেছিল" ✅

মডেল কোটি কোটি বাক্য পড়ে
পরবর্তী শব্দ অনুমান করতে শেখে।
```

### আর্কিটেকচার: ট্রান্সফর্মার (ডিকোডার ওনলি)

```
┌─────────────────────────────────┐
│         ট্রান্সফর্মার            │
│                                 │
│  ┌───────────────────────┐      │
│  │   ইনপুট এম্বেডিং      │      │
│  └──────────┬────────────┘      │
│             ▼                   │
│  ┌───────────────────────┐      │
│  │ মাল্টি-হেড অ্যাটেনশন  │──┐   │
│  └──────────┬────────────┘  │   │
│             ▼               │   │
│  ┌───────────────────────┐  │   │
│  │   ফিড ফরোয়ার্ড লেয়ার │  │ x৯৬ লেয়ার (GPT-4)
│  └──────────┬────────────┘  │   │
│             ▼               │   │
│  ┌───────────────────────┐  │   │
│  │   লেয়ার নরমালাইজেশন  │──┘   │
│  └──────────┬────────────┘      │
│             ▼                   │
│  ┌───────────────────────┐      │
│  │   আউটপুট প্রেডিকশন    │      │
│  └───────────────────────┘      │
└─────────────────────────────────┘
```

### ট্রেইনিং খরচ:

| মডেল | ব্যবহৃত GPU | ট্রেইনিং সময় | খরচ |
|------|-------------|---------------|------|
| GPT-3 | ১০,০০০ V100 | ~৩৪ দিন | ~৫ মিলিয়ন ডলার |
| GPT-4 | ২৫,০০০ A100 | ~৯০ দিন | ~১০০ মিলিয়ন ডলার |
| LLaMA 3 (405B) | ১৬,০০০ H100 | ~৫৪ দিন | ~৩০ মিলিয়ন ডলার |

### প্রি-ট্রেইনিংয়ের মূল ধারণাগুলো:

```
📌 সেলফ-অ্যাটেনশন মেকানিজম
   - মডেল শেখে কোন শব্দগুলো একে অপরের সাথে সম্পর্কিত
   
   "প্রাণীটি রাস্তা পার হয়নি কারণ সে খুব ক্লান্ত ছিল"
                                        │
                          "সে" বোঝায় → "প্রাণীটি" ✅ ("রাস্তা" নয়)

📌 লস ফাংশন: ক্রস-এনট্রপি লস
   - মডেলের ভুল অনুমান কতটা ভুল তা পরিমাপ করে
   - লক্ষ্য: এই লস কমানো

📌 অপ্টিমাইজার: Adam / AdamW
   - ভুল কমাতে মডেলের ওজন আপডেট করে

📌 লার্নিং রেট শিডিউলার
   - মডেল কত দ্রুত শিখবে তা নিয়ন্ত্রণ করে
   - ওয়ার্ম-আপ → শীর্ষ → হ্রাস
```

## ধাপ ৪: সুপারভাইজড ফাইন-টিউনিং (SFT)

```
প্রি-ট্রেইনড মডেল → মানুষের লেখা প্রশ্ন-উত্তর দিয়ে ফাইন-টিউন → উন্নত মডেল
```

### এটা কিভাবে কাজ করে:

```
মানুষের লেখা উদাহরণ:

  প্রশ্ন: "মাধ্যাকর্ষণ সহজ ভাষায় ব্যাখ্যা করো"
  উত্তর: "মাধ্যাকর্ষণ হলো এমন একটি শক্তি যা বস্তুগুলোকে 
          একে অপরের দিকে টানে। এই কারণেই জিনিসপত্র
          মাটিতে পড়ে..."

  প্রশ্ন: "দুটি সংখ্যা যোগ করার Python ফাংশন লেখো"
  উত্তর: "def add(a, b):
               return a + b"

হাজার হাজার এরকম উদাহরণ মানব লেবেলাররা তৈরি করে
```

### ফাইন-টিউনিংয়ের আগে বনাম পরে:

```
❌ আগে (বেস মডেল):
   ব্যবহারকারী: "ফ্রান্সের রাজধানী কী?"
   মডেল: "জার্মানির রাজধানী কী? ইতালির 
          রাজধানী কী?..."  (শুধু প্যাটার্ন চালিয়ে যায়)

✅ পরে (ফাইন-টিউনড মডেল):
   ব্যবহারকারী: "ফ্রান্সের রাজধানী কী?"
   মডেল: "ফ্রান্সের রাজধানী প্যারিস।" ✅
```

## ধাপ ৫: RLHF (মানুষের প্রতিক্রিয়া থেকে রিইনফোর্সমেন্ট লার্নিং)

### এটাই ChatGPT-কে সত্যিকারের সহায়ক করে তোলে

```
┌──────────────────────────────────────────┐
│              RLHF পাইপলাইন               │
│                                          │
│  ধাপ ক: মানুষের র‍্যাংকিং সংগ্রহ        │
│  ধাপ খ: রিওয়ার্ড মডেল ট্রেইন করা        │
│  ধাপ গ: PPO দিয়ে অপ্টিমাইজ করা          │
└──────────────────────────────────────────┘
```

### ধাপ ক: মানুষের র‍্যাংকিং

```
প্রশ্ন: "কোয়ান্টাম কম্পিউটিং ব্যাখ্যা করো"

উত্তর A: "কোয়ান্টাম কম্পিউটিং কিউবিট ব্যবহার করে..." (ভালো ✅)
উত্তর B: "কম্পিউটিং একটি ক্ষেত্র..."                  (অস্পষ্ট ❌)
উত্তর C: "আমি জানি না"                                (খারাপ ❌❌)

মানব র‍্যাংকার বলে: A > B > C
```

### ধাপ খ: রিওয়ার্ড মডেল

```
┌─────────────┐     ┌──────────────────┐
│  প্রশ্ন +   │────►│  রিওয়ার্ড মডেল   │────► স্কোর: ০.৮৫
│  উত্তর      │     │  (মানুষের রেটিং  │
│             │     │   থেকে শেখা)     │
└─────────────┘     └──────────────────┘

ভালো উত্তর  → উচ্চ স্কোর  (০.৯)
খারাপ উত্তর → নিম্ন স্কোর (০.১)
```

### ধাপ গ: PPO (Proximal Policy Optimization)

```
┌─────────────────────────────────────────┐
│                                         │
│   LLM উত্তর তৈরি করে                    │
│          ▼                              │
│   রিওয়ার্ড মডেল সেটাকে স্কোর দেয়        │
│          ▼                              │
│   PPO মডেলের ওজন আপডেট করে             │
│   যাতে আরও ভালো উত্তর তৈরি হয়          │
│          ▼                              │
│   হাজার হাজার বার এটা পুনরাবৃত্তি হয়     │
│                                         │
└─────────────────────────────────────────┘
```

## 🔑 মূল পার্থক্য: GPT-4 বনাম LLaMA 3

| বৈশিষ্ট্য | GPT-4 (OpenAI) | LLaMA 3 (Meta) |
|-----------|----------------|-----------------|
| ওপেন সোর্স | ❌ না | ✅ হ্যাঁ |
| প্যারামিটার | ~১.৮ ট্রিলিয়ন (আনুমানিক) | 8B / 70B / 405B |

| টোকেনাইজার | BPE (tiktoken) | SentencePiece |
| আর্কিটেকচার | Mixture of Experts (MoE) | Dense Transformer |
| RLHF | ✅ PPO | ✅ DPO |
| অ্যাক্সেস | শুধু API | ওজন ডাউনলোড করা যায় |
| ট্রেইনিং খরচ | ~১০০ মিলিয়ন ডলার | ~৩০ মিলিয়ন ডলার |

```

⭐ RLHF           = মানুষের কাছ থেকে ফিডব্যাক নেওয়া - কোন উত্তরটা ভালো

সব মিলিয়ে = একটি জ্ঞানী, সহায়ক AI সহকারী!
```
