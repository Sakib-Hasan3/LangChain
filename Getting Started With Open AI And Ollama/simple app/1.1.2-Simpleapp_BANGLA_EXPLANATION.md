## সেল ৬: ডকুমেন্টগুলি চাংকে বিভক্ত করুন
**ধরন:** Python কোড
```python
### Load Data--> Docs-->Divide our Docuemnts into chunks dcouments-->text-->vectors-->Vector Embeddings--->Vector Store DB
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter=RecursiveCharacterTextSplitter(chunk_size=1000,chunk_overlap=200)
documents=text_splitter.split_documents(docs)
```
**ব্যাখ্যা:**
- ডকুমেন্টগুলি পরিচালনাযোগ্য চাংকে ভাগ করতে `RecursiveCharacterTextSplitter` ইম্পোর্ট করে
- **চাংক সাইজ:** প্রতিটি চাংকে ১০০০ অক্ষর
- **ওভারল্যাপ:** চাংকের মধ্যে ২০০ অক্ষর (প্রসঙ্গ ধারাবাহিকতা বজায় রাখে)
- সমস্ত লোড করা ডকুমেন্টগুলি বিভক্ত করে এবং চাংকগুলি `documents` ভেরিয়েবলে সংরক্ষণ করে
- এই চাংকিং প্রয়োজনীয় কারণ LLM-এর টোকেন সীমা রয়েছে এবং ভেক্টর ডাটাবেসগুলি ছোট, সিমান্টিক ইউনিটগুলির সাথে ভালোভাবে কাজ করে

## সেল ৯: FAISS দিয়ে ভেক্টর স্টোর তৈরি করুন
**ধরন:** Python কোড
```python
from langchain_community.vectorstores import FAISS
vectorstoredb=FAISS.from_documents(documents,embeddings)
```
**ব্যাখ্যা:**
- FAISS (Facebook AI Similarity Search) ভেক্টর ডাটাবেস ইম্পোর্ট করে
- নিম্নলিখিত কাজ করে ভেক্টর স্টোর তৈরি করে:
  1. OpenAI এমবেডিংস ব্যবহার করে সমস্ত ডকুমেন্ট চাংক এম্বেড করে
  2. এমবেডিংগুলি দক্ষ সিমিলারিটি সার্চের জন্য FAISS এ সংরক্ষণ করে
- `vectorstoredb` এখন সিমান্টিক সিমিলারিটির উপর ভিত্তি করে প্রাসঙ্গিক ডকুমেন্ট রিট্রিভ করার জন্য প্রস্তুত
- **নোট:** OpenAI কোটা সীমার কারণে এই সেলটি ব্যর্থ হয় (বিলিং প্রয়োজন)

## সেল ১৩: প্রম্পট টেমপ্লেট সহ ডকুমেন্ট চেইন তৈরি করুন
**ধরন:** Python কোড
```python
## Retrieval Chain, Document chain

from langchain.chains.combine_documents import create_stuff_documents_chain
from langchain_core.prompts import ChatPromptTemplate

prompt=ChatPromptTemplate.from_template(
    """
Answer the following question based only on the provided context:
<context>
{context}
</context>


"""
)

document_chain=create_stuff_documents_chain(llm,prompt)
document_chain
```
**ব্যাখ্যা:**
- একটি **ডকুমেন্ট চেইন** তৈরি করে যা:
  1. একটি প্রশ্ন এবং প্রসঙ্গ ডকুমেন্টগুলি নেয়
  2. "stuff" পদ্ধতি ব্যবহার করে ডকুমেন্টগুলি একত্রিত করে (সমস্ত ডকস কনক্যাটেনেট করে)
  3. LLM-কে নির্দেশ দিতে প্রম্পট টেমপ্লেট ব্যবহার করে
- **প্রম্পট স্ট্রাকচার:** LLM-কে শুধুমাত্র প্রদত্ত প্রসঙ্গের উপর ভিত্তি করে উত্তর দিতে নির্দেশ দেয়
- `create_stuff_documents_chain()` প্রম্পটে ডকুমেন্টগুলি একীভূত করার কাজ পরিচালনা করে
- এই চেইন নিশ্চিত করে যে উত্তরগুলি প্রদত্ত ডকুমেন্টগুলিতে ভিত্তি করে রয়েছে

## সেল ১৭: রিট্রিভাল চেইন তৈরি করুন
**ধরন:** Python কোড
```python
retriever=vectorstoredb.as_retriever()
from langchain.chains import create_retrieval_chain
retrieval_chain=create_retrieval_chain(retriever,document_chain)

```
**ব্যাখ্যা:**
- `as_retriever()` ব্যবহার করে ভেক্টর স্টোরকে রিট্রিভারে রূপান্তরিত করে
- একটি **রিট্রিভাল চেইন** তৈরি করে যা:
  1. ব্যবহারকারীর কোয়েরি ইনপুট হিসাবে নেয়
  2. ভেক্টর স্টোর থেকে প্রাসঙ্গিক ডকুমেন্ট খুঁজতে রিট্রিভার ব্যবহার করে
  3. এই ডকুমেন্টগুলি ডকুমেন্ট চেইনে পাস করে
  4. রিট্রিভ করা প্রসঙ্গের উপর ভিত্তি করে উত্তর জেনারেট করে
- এটি সম্পূর্ণ **RAG (রিট্রিভাল-অগমেন্টেড জেনারেশন)** পাইপলাইন

retrieval_chain

  4. LLM-জেনারেটেড উত্তর রিটার্ন করে

## পাইপলাইন ওভারভিউ

নোটবুকটি একটি সম্পূর্ণ **RAG (রিট্রিভাল-অগমেন্টেড জেনারেশন)** পাইপলাইন প্রদর্শন করে:

```
১. ডেটা ইনজেশন: WebBaseLoader → ওয়েব পেজ স্ক্র্যাপ করুন
২. টেক্সট স্প্লিটিং: ডকুমেন্টগুলি ওভারল্যাপিং চাংকে বিভক্ত করুন
३. এমবেডিংস: টেক্সট চাংকগুলিকে ভেক্টরে রূপান্তরিত করুন (OpenAI এমবেডিংস)
४. ভেক্টর স্টোর: সিমিলারিটি সার্চের জন্য FAISS এ এমবেডিংস সংরক্ষণ করুন
५. রিট্রিভার: প্রাসঙ্গিক ডকুমেন্ট খুঁজতে ভেক্টর স্টোর কোয়েরি করুন
६. ডকুমেন্ট চেইন: ডকুমেন্টগুলি একত্রিত করুন এবং প্রম্পট তৈরি করুন
७. LLM: রিট্রিভ করা প্রসঙ্গের উপর ভিত্তি করে উত্তর জেনারেট করুন (GPT-4o)
८. রেসপন্স: ব্যবহারকারীকে উত্তর + প্রসঙ্গ রিটার্ন করুন
```

**প্রধান সুবিধা:**
- প্রকৃত ডকুমেন্টের উপর ভিত্তি করে প্রতিক্রিয়া (হ্যালুসিনেশন হ্রাস করে)
- ডায়নামিক রিট্রিভাল (প্রতিটি কোয়েরির জন্য প্রাসঙ্গিক ডকস পরিবর্তন হয়)
- স্কেলেবল (বড় ডকুমেন্ট কালেকশন পরিচালনা করতে পারে)
- স্বচ্ছ (দেখতে পারেন কোন ডকুমেন্টগুলি ব্যবহার করা হয়েছে)
