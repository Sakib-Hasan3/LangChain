## 3. **Preparing the Data for MySQL Database** 

---

# 🔥 Brutal Truth

SQLite tolerate করে:

* sloppy schema
* inconsistent data
* small datasets

👉 MySQL করবে না।

> **MySQL-এ ভুল data design = slow queries + wrong joins + broken agent**

---

# 🔷 1. SQLite vs MySQL — mindset shift

| বিষয়           | SQLite | MySQL        |
| -------------- | ------ | ------------ |
| setup          | simple | server-based |
| scale          | small  | large        |
| strictness     | low    | high         |
| production use | rare   | common       |

👉 MySQL = production-ready discipline দরকার

---

```text

```

# 🔷 3. Step-by-step Breakdown

---

## ✅ Step 1: Data Cleaning (More Strict)

MySQL-এ sloppy data চলবে না।

Check করো:

* NULL values
* duplicate rows
* data type mismatch
* inconsistent format

---

### ❌ Example (bad data)

| id | amount      |
| -- | ----------- |
| 1  | 100         |
| 2  | one hundred |

👉 MySQL:

* error দিবে বা wrong insert হবে

---

## ✅ Step 2: Schema Design (Critical Upgrade)

SQLite-এ loose typing allowed
👉 MySQL-এ strict typing দরকার

---

### Example Schema

```sql
CREATE TABLE customers (
    customer_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    country VARCHAR(50)
);
```

---

```sql
CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    amount DECIMAL(10,2),
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);
```

---

### 🧠 Difference from SQLite

* `INT`, `VARCHAR`, `DECIMAL`
* strict types
* constraints enforced

# 🔷 7. CSV → MySQL

```python
import pandas as pd
import mysql.connector

df = pd.read_csv("data.csv")

conn = mysql.connector.connect(
    host="localhost",
    user="root",
    password="your_password",
    database="sales_db"
)

cursor = conn.cursor()

for _, row in df.iterrows():
    cursor.execute(
        "INSERT INTO customers (name, country) VALUES (%s, %s)",
        (row['name'], row['country'])
    )

conn.commit()
```

# 🔷 8. Indexing (Critical for Performance)

SQLite-এ ignore করলেও চলে
👉 MySQL-এ ignore করলে slow হবে

---

```sql
CREATE INDEX idx_customer_id ON orders(customer_id);
```

---

## 🧠 কেন দরকার?

Agent query করবে:

```sql
SELECT * FROM orders WHERE customer_id = 5;
```

👉 index না থাকলে full table scan

# 🔷 9. Data Validation

---

### Basic check

```sql
SELECT COUNT(*) FROM customers;
```

---

### Join check

```sql
SELECT c.name, o.amount
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id;
```

---

### Aggregation test

```sql
SELECT country, COUNT(*) FROM customers GROUP BY country;
```

# 🔷 10. SQL Agent-এর জন্য Optimization

---

## ✅ Clean Naming

❌ Bad:

* `tbl1`
* `data_final_new`

✅ Good:

* `customers`
* `orders`
* `sales`

---

## ✅ Consistent Columns

* `customer_id` everywhere
* not `cust_id`, `id`, `cid`

---

## ✅ Proper Relationships

* foreign key must
* join clarity

---

## ✅ Avoid Over-Normalization

too many tables = agent confused

---

# 🔷 11. Real Query Example

User:

> “Which customer spent the most?”

```sql
SELECT c.name, SUM(o.amount) AS total_spent
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
GROUP BY c.name
ORDER BY total_spent DESC
LIMIT 1;
```

👉 এটা work করবে only if:

* schema clean
* relation correct
* data consistent
