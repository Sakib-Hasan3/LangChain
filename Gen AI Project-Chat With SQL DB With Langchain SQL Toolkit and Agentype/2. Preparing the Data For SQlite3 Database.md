# 🔷 1. “Preparing Data” আসলে কী?

> **Raw data → structured relational database (SQLite)**

* table design করা

* SQLite-এ load করা

## ✅ Step 3: Schema Design (Critical)

> **Table structure ঠিক না হলে agent fail করবে**

---

### Example: Sales DB

#### customers table

```sql
CREATE TABLE customers (
    customer_id INTEGER PRIMARY KEY,
    name TEXT,
    country TEXT
);
```

---

#### orders table

```sql
CREATE TABLE orders (
    order_id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    amount REAL,
    date TEXT,
    FOREIGN KEY(customer_id) REFERENCES customers(customer_id)
);
```

---

### 🧠 কেন important?

Agent বুঝতে পারবে:

* customer → order relation
* join কিভাবে করতে হবে

---

# 🔷 5. SQLite Database তৈরি করা

### Python দিয়ে

```python
import sqlite3

conn = sqlite3.connect("my_database.db")
cursor = conn.cursor()
```

---

### Table create

```python
cursor.execute("""
CREATE TABLE customers (
    id INTEGER PRIMARY KEY,
    name TEXT,
    country TEXT
)
""")
```

---

# 🔷 6. Data insert করা

```python
cursor.execute("INSERT INTO customers (name, country) VALUES (?, ?)", ("Rahim", "Bangladesh"))
conn.commit()
```

---

# 🔷 7. CSV → SQLite

```python
import pandas as pd

df = pd.read_csv("data.csv")

conn = sqlite3.connect("my_database.db")
df.to_sql("customers", conn, if_exists="replace", index=False)
```

# 🔷 9. SQL Agent-এর জন্য Data Optimization

---

## 🧠 Naming Rules

# 🔷 10. Common Mistakes

---

## ❌ Flat table ব্যবহার করা

সব data এক table-এ রাখলে:

* join impossible
* analysis weak

---

## ❌ No relationships

foreign key না দিলে:

* agent বুঝবে না relation

---

## ❌ Dirty data

* null
* duplicates
* mixed types

---

## ❌ Over-complex schema

too many tables → agent confused

---

# 🔷 11. Real Example Query

User:

> “Which customer has highest spending?”

Agent needs:

* customers table
* orders table
* join logic

```sql
SELECT customers.name, SUM(orders.amount) as total
FROM customers
JOIN orders ON customers.customer_id = orders.customer_id
GROUP BY customers.name
ORDER BY total DESC
LIMIT 1;
```

👉 এই query possible only if schema clean
